# 自定义语言包指南

本文档详细介绍了如何创建和使用自定义语言包，以便在 RemixWarp 编辑器中添加新的语言支持。

## 语言包格式

自定义语言包必须是一个 JSON 文件，包含以下字段：

### 基本结构

```json
{
  "locale": "语言代码",
  "name": "语言名称",
  "parentLocale": "父语言代码（可选）",
  "messages": {
    "消息ID": "翻译文本",
    ...
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `locale` | 字符串 | 是 | 语言代码，遵循 BCP 47 标准，如 `zh-cn`、`en-us` 等 |
| `name` | 字符串 | 是 | 语言名称，使用该语言自身的书写方式，如中文的 "简体中文" |
| `parentLocale` | 字符串 | 否 | 父语言代码，用于继承父语言的翻译 |
| `messages` | 对象 | 是 | 翻译消息的键值对，键为消息ID，值为翻译文本 |

## 消息ID格式

消息ID通常遵循以下格式：

```
模块.组件.消息描述
```

例如：
- `gui.menuBar.file` - 菜单栏文件菜单
- `gui.editor.stage` - 编辑器舞台
- `blocks.control.if` - 控制积木的如果语句

## 示例语言包

以下是一个完整的语言包示例：

```json
{
  "locale": "zh-cn",
  "name": "简体中文",
  "parentLocale": "zh",
  "messages": {
    "gui.menuBar.file": "文件",
    "gui.menuBar.edit": "编辑",
    "gui.menuBar.help": "帮助",
    "gui.editor.stage": "舞台",
    "gui.editor.sprites": "角色",
    "blocks.control.if": "如果",
    "blocks.control.then": "那么",
    "blocks.control.else": "否则",
    "blocks.motion.move": "移动",
    "blocks.looks.say": "说",
    "blocks.sound.play": "播放"
  }
}
```

## 如何导入语言包

1. 在 RemixWarp 编辑器中，点击顶部菜单栏的 "设置" 菜单
2. 选择 "语言" 子菜单
3. 点击 "导入自定义语言包" 按钮
4. 在弹出的文件选择对话框中，选择您创建的语言包 JSON 文件
5. 等待导入完成，系统会显示成功消息
6. 导入成功后，您可以在语言列表中选择新导入的语言

## 注意事项

1. 语言包文件必须是有效的 JSON 格式
2. 语言代码必须遵循 BCP 47 标准
3. 消息ID应与编辑器中使用的ID保持一致
4. 导入的语言包会临时存储在内存中，刷新页面后需要重新导入
5. 为了获得最佳效果，建议翻译所有常用的消息ID

## 常见问题

### Q: 导入语言包后看不到新语言

A: 请检查语言包格式是否正确，特别是 `locale` 和 `name` 字段是否填写正确。

### Q: 翻译不生效

A: 请确保消息ID与编辑器中使用的ID完全一致，大小写也需要匹配。

### Q: 语言包文件太大导致导入失败

A: 建议将语言包分成多个小文件，或者只翻译最常用的消息。

## 高级功能

### 占位符

翻译文本中可以使用占位符，例如：

```json
{
  "messages": {
    "gui.project.saveAs": "另存为 {name}"
  }
}
```

### 复数形式

对于需要支持复数形式的语言，可以使用特殊的复数规则，例如：

```json
{
  "messages": {
    "gui.project.items": {
      "one": "{count} 个项目",
      "other": "{count} 个项目"
    }
  }
}
```
